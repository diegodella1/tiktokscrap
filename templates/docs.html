Documentación del monitor

Aplicación Flask para monitorear cuentas de TikTok, persistir posts recientes en SQLite y exponer un dashboard con autenticación y API JSON.

Scraper
yt-dlp sobre perfiles de TikTok
Persistencia
SQLite local en data/tiktok.db
Scheduler
Auto-scan periódico con APScheduler

Variables de entorno

Configuración mínima recomendada:

MONITOR_PASSWORD=tu_password
SCAN_INTERVAL_MINUTES=30
SLACK_WEBHOOK_URL=

Notas

  • Si MONITOR_PASSWORD está vacío, la app queda sin login.
  • SCAN_INTERVAL_MINUTES controla el job automático de TikTok.
  • SLACK_WEBHOOK_URL es opcional y solo se usa para avisar posts nuevos.

Ejecución

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python app.py

La app escucha por defecto en http://localhost:3457.

Endpoints

GET/api/accounts
Lista las cuentas activas monitoreadas.
POST/api/accounts
Agrega una cuenta. Body: {"username":"usuario"}.
DELETE/api/accounts/<username>
Elimina la cuenta y sus posts asociados.
GET/api/posts
Lista posts. Soporta username, limit y offset.
GET/api/posts/counts
Devuelve cantidad de posts por cuenta y el total.
POST/api/scan
Dispara un scan manual inmediato.
GET/api/status
Estado del scheduler, último scan y progreso si hay uno corriendo.
POST/api/autoscan
Activa o pausa el job automático. Body: {"enabled":true}.
POST/api/interval
Cambia el intervalo del scheduler. Body: {"minutes":30}.
