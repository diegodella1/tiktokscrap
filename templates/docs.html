Que es Social Monitor

Sistema de monitoreo que rastrea cuentas de TikTok e Instagram y detecta publicaciones nuevas automaticamente. Muestra la URL, descripcion, stats y thumbnail de cada video/reel en un dashboard web con tabs separados por plataforma.

No descarga videos ni interactua con las plataformas. Solo lee metadata publica.

Como funciona

Ciclo de escaneo TikTok

  1. Cada X minutos (configurable) se ejecuta un scan automatico.
  2. Por cada cuenta activa, se extraen los ultimos ~30 posts usando yt-dlp (solo metadata, sin descargar video).
  3. Se compara contra la base de datos. Los posts nuevos se guardan.
  4. Los posts con mas de 24 horas de antiguedad se eliminan automaticamente (ventana rolling).
  5. Si una cuenta falla (ej: cuenta privada, error de red), se loguea el error y se continua con la siguiente.

Ciclo de escaneo Instagram

  1. Scheduler independiente con su propio intervalo (default 30 min).
  2. Por cada cuenta IG activa, se extraen los ultimos ~20 posts usando instaloader con sesion autenticada. Solo se guardan Reels (videos).
  3. Se hace upsert: posts nuevos se insertan, stats de existentes se actualizan.
  4. Los reels con mas de 24 horas se purgan igual que TikTok.
  5. Pausa de 10 segundos entre perfiles para evitar rate limiting.
  6. Si instaloader pide esperar mas de 120 seg, se aborta ese perfil y se continua.

Ventana de 24 horas

Solo se retienen posts descubiertos en las ultimas 24 horas (ambas plataformas). Despues de cada scan los mas viejos se purgan automaticamente.

Instagram Reels

Requisitos

Instagram bloquea requests sin autenticar (429 inmediato). Por eso se requiere una sesion de instaloader generada con credenciales reales.

Setup inicial (una sola vez)

# 1. Activar el venv
cd /home/diego/Documents/tiktokscrap
source venv/bin/activate

# 2. Generar session file (pide password por terminal)
instaloader --login TU_USUARIO_IG

# 3. Configurar en .env
IG_SESSION_USER=TU_USUARIO_IG

# 4. Reiniciar servicio
sudo systemctl restart tiktok-monitor
Nota La sesion se guarda en ~/.config/instaloader/session-USERNAME. Si Instagram invalida la sesion (cambio de password, sospecha de bot, etc.), hay que regenerarla corriendo instaloader --login USERNAME de nuevo.

Que scrapea

  • Solo Reels (posts de video). Fotos y carruseles se ignoran.
  • De cada reel: shortcode, caption, fecha, views, likes, comments, thumbnail.
  • Solo cuentas publicas (o que siga el usuario de la sesion).

Guia de uso

Login

Al acceder por primera vez se pide un password. Se guarda en una cookie por 30 dias, no hace falta loguearse cada vez.

Tabs: TikTok e Instagram

El dashboard tiene dos tabs arriba: TikTok (por defecto) e Instagram. Cada tab tiene su propia lista de cuentas, scanner independiente, timeline y filtros. Funcionan de forma identica pero contra plataformas distintas.

Agregar cuentas

  1. Selecciona el tab de la plataforma (TikTok o Instagram).
  2. En la seccion Cuentas, escribi el username (con o sin @).
  3. Click en Agregar o presiona Enter.
  4. La cuenta aparece como chip. Se puede agregar hasta ~10 cuentas por plataforma.
Nota El username se normaliza: se quita el @, se pasa a minusculas. Ejemplo: @Khaby.Lame se guarda como khaby.lame.

Eliminar cuentas

Click en la x del chip. Se elimina la cuenta y todos sus posts/reels de la base de datos (cascade).

Escanear

  • Automatico: cada plataforma tiene su propio scheduler con intervalo independiente. El toggle Auto ON/OFF y el intervalo se configuran por separado en cada tab.
  • Manual: click en Escanear ahora dentro del tab correspondiente.

Cambiar intervalo de escaneo

En cada tab, al lado del boton de escanear, hay un input numerico que dice cada [30] min. Cambia el numero y se aplica al instante. Rango: 1 a 1440 minutos (24 horas). Cada plataforma tiene su propio intervalo.

Filtrar posts

Arriba del feed hay chips de filtro: Todos y uno por cada cuenta. Click para filtrar. El feed muestra los posts mas recientes primero.

Ver posts

Cada post/reel muestra:

  • @username del autor con avatar
  • Horarios en 4 zonas (ver abajo)
  • Descripcion/caption completa (hashtags, etc.)
  • Stats: views, likes, comments (y reposts en TikTok)
  • Thumbnail del video
  • Link directo al video en la plataforma

Metricas del topbar

La barra superior muestra resumen de ambas plataformas: ultimo scan, cantidad de cuentas y posts para TikTok (TT) e Instagram (IG).

Ejemplo con curl

# Login y guardar cookie
curl -c cookies.txt -d "password=TU_PASSWORD" -L https://tiktok.diegodella.ar/login

# Agregar cuenta
curl -b cookies.txt -X POST https://tiktok.diegodella.ar/api/accounts \
  -H "Content-Type: application/json" \
  -d '{"username": "khaby.lame"}'

# Scan manual
curl -b cookies.txt -X POST https://tiktok.diegodella.ar/api/scan

# Ver posts
curl -b cookies.txt https://tiktok.diegodella.ar/api/posts?limit=10

Administracion

Servicio systemd

El monitor corre como servicio del sistema. Se inicia automaticamente al prender la maquina.

# Ver estado
sudo systemctl status tiktok-monitor

# Reiniciar
sudo systemctl restart tiktok-monitor

# Ver logs en vivo
sudo journalctl -u tiktok-monitor -f

# Parar
sudo systemctl stop tiktok-monitor

Limites y consideraciones

TikTok

  • ~10 cuentas recomendado. Cada cuenta tarda ~3 seg. Con 10 cuentas, un scan toma ~30 seg.
  • Solo cuentas publicas. Cuentas privadas dan error (se loguea y se sigue).
  • No descarga video. Solo extrae metadata.
  • Rate limiting. 3 segundos de pausa entre cuentas.

Instagram

  • Sesion obligatoria. Sin IG_SESSION_USER configurado, el scan IG falla con 429.
  • Solo Reels (video). Fotos y carruseles se ignoran.
  • ~10 cuentas recomendado. Cada cuenta tarda ~10 seg (pausa mas larga). Con 10 cuentas, un scan toma ~2 min.
  • Rate limiting estricto. Si instaloader pide esperar mas de 120 seg, se aborta ese perfil. Instagram es mas agresivo que TikTok con limits.
  • Cuentas privadas: solo funcionan si el usuario de la sesion las sigue.

General

  • Retencion 24hs. Los posts/reels se purgan automaticamente. No es un archivo historico.
  • Sin notificaciones push. Es un dashboard de consulta.
  • Puerto 3457. Acceso local en http://192.168.1.14:3457, publico en https://tiktok.diegodella.ar.

Troubleshooting

Instagram: scan devuelve 0 reels

Causa mas comun: rate limit (429). Instagram bloquea la sesion temporalmente si se hacen muchas requests. El bloqueo dura ~30 minutos.

Que hacer:

  1. Esperar 30 minutos sin hacer scans manuales.
  2. No agregar muchas cuentas IG de golpe (cada add no hace API call, pero el primer scan si hace 1 request por cuenta).
  3. Revisar logs: tail -50 /tmp/tiktokscrap.log | grep -i "rate\|429\|ig"
  4. Si el log dice likely rate-limited, confirma que es un bloqueo temporal.

Instagram: "No session file found"

La sesion de instaloader no existe o fue borrada. Hay que regenerarla: 

cd /home/diego/Documents/tiktokscrap
source venv/bin/activate
instaloader --login TU_USUARIO_IG
# Ingresa password, luego reiniciar servicio
sudo systemctl restart tiktok-monitor
Importante Este comando se corre desde una terminal interactiva (SSH o consola), no desde la UI web. Pide el password por teclado.

Instagram: sesion invalidada

Causas posibles:

  • Cambiaste la password de Instagram.
  • Instagram detecto actividad sospechosa y forzo logout.
  • Sesion expirada por tiempo (raro, pero pasa).

Solucion: regenerar sesion con instaloader --login USERNAME. Si Instagram pide verificacion 2FA, seguir las instrucciones en terminal.

Instagram: cuentas privadas

Solo se pueden scrapear cuentas que sean publicas o que el usuario de la sesion (IG_SESSION_USER) siga. Si una cuenta es privada y no la seguis, el log dira private profile y se saltea.

TikTok: scan no trae posts

TikTok cambia su estructura seguido. Si deja de funcionar: 

source venv/bin/activate
pip install -U yt-dlp
sudo systemctl restart tiktok-monitor

El dashboard carga pero no muestra datos

  • Verifica que el servicio este corriendo: sudo systemctl status tiktok-monitor
  • Verifica logs: tail -20 /tmp/tiktokscrap.log
  • Verifica que haya cuentas activas en el tab correspondiente.
  • Ejecuta un scan manual desde el boton "Escanear ahora".

Cuantas cuentas IG se pueden monitorear?

Recomendado: maximo 10 cuentas. Con 10 cuentas y 10 seg de pausa entre cada una, un scan toma ~2 minutos. Instagram tiene limites estrictos; muchas cuentas aumentan la chance de 429. El intervalo minimo recomendado para IG es 30 minutos.

Se puede usar sin sesion de Instagram?

No. Sin IG_SESSION_USER configurado, Instagram bloquea con 429 en la primera request. La sesion autenticada es obligatoria. TikTok en cambio funciona sin autenticacion.
